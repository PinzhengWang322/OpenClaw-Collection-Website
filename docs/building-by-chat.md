# Building The Site By Chat

这份公开版把原始对话流程整理成了一个更容易复用的版本。核心不是逐字复刻聊天记录，而是把“怎么从一句模糊需求滚成一个完整站点”拆清楚。

## Phase 1: Start From A Broad Prompt

一开始的需求很宽：

- 在家目录下新建一个项目
- 做一个私人收藏网站
- 三大模块：影视、游戏、去过的地方
- 卡片式布局、支持分页、支持详情页
- 数据先用 JSON 和图片文件

这个阶段最重要的是先把骨架搭出来，而不是先抠所有字段和视觉细节。

## Phase 2: Keep Tightening The Frontend

接下来的大量对话都不是在“推翻重做”，而是在连续微调：

- 去掉不必要的数据面板和数据库展示
- 把背景改成更轻、更可爱的蓝色
- 调整一行卡片数量和卡片高度
- 改 Hero 文案和歌词排版
- 修正移动端的卡片比例
- 把桌面端稳定到 `6 x 2`

这类项目很适合“连续小步改动”的对话方式，因为每次只改一处，视觉反馈很快。

## Phase 3: Turn It From A Static Demo Into A Usable Tool

当页面样式差不多稳定之后，重点开始从“好看”转到“好用”：

- 增加网页端管理面板
- 加新增 / 编辑 / 删除 / 追加图片接口
- 统一按 `YY.MM` 自动倒序排序
- 同步生成 SQLite 镜像
- 补 README，写清楚命令行和网页端怎么维护数据

到这里，这个站点已经不只是静态页面，而是一个真正能维护内容的小系统了。

## Phase 4: Connect OpenClaw

站点本身跑起来以后，再把 OpenClaw 接进来：

- 用户在 Discord 里发自然语言请求
- OpenClaw agent 理解“这是新增卡片 / 改字段 / 删除卡片 / 换封面 / 追加详情图”
- agent 调用本地 CLI 或站点写接口
- 成功后回一句简短确认

这样一来，网站就从“自己去后台点按钮”变成了“聊天驱动的内容管理”。

## Phase 5: Sanitize It For Public Release

最后一步其实很关键：一个能跑的私人原型，不等于可以直接公开。

公开前需要主动做几层处理：

- 把人名、昵称、私人称呼全部换成 demo 文案
- 把真实旅行/生日/演唱会等生活内容裁掉
- 不暴露 token、IP、频道 ID、绝对路径
- 不带上私人图片和聊天截图原件
- 把 README 改成别人第一次看到也能理解的 showcase 文档

这也是为什么这个 repo 不是直接拷贝私人站点，而是“重新包装后的公开版”。
